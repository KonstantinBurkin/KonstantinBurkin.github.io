Introduction

This work was created and submitted as a final course project for MSU course "Machine learning for applied problems" in spring semester 2022. The project was written in Google Colab, using Python version 3.7.13. This work is available in my Github repository, where it is possible to download Colab Notebook and see the code.

The goal is to create Machine Learning (ML) model that predicts the number of sales for each product in every store in Delivery Club App for the next week. The dataframe describing Delivery Club sales from 160 stores in 10 russian cities was provided by the supervisor. The dataframe contained information abouth the date of sales, weather conditions, product types, stores, and their locations.

The result of this project is a dataframe containing id of products and corresponding number of predicted sales. The error of predicted values is calculated as mean absotute error and should not exceede more than 4.10 units. The accuracy of predictions was evaluated using Yandex.contest platform.

Brief outline

  • Data import, setting up notebook environment: import of libraries and conversion of dataframe to English format.
  • Exploration of dataframes: description of data types, number of unique values, description of column names, calculation of NAs. Statistical evaluation of numeric columns, initial correlation between columns.
  • Visualisation of different columns and subsets from dataframe. Looking for patterns between sales column and other columns.
  • Transform data for ML models. Divide data into several parts for training and evaluating ML models and buliding predictions for testing part.
  • Compare diferent ML models and choose the best one with minimal absolute error.
  • Build predictions and visualize the result.

Data visualisation

All of the graphs presented in this work are interactive. It is possible to choose one out of many lines shown on the graph and choose the interval across both axes to explore the data. To go back to initial state of the graph it is necessary to double-click the graph.

The graph below shows the number of sales during 7 month period in 10 cities. As seen from the graph the number of sales changes through the week in each city. In different cities there is a different rate of sales. In larger cities, like Moscow, the sales rate is the highest. The noticeable sharp decrease in sales can be observed around New Year, which is an important national holiday in Russia, during which people generally stay at home with their families and most of the stores are closed. The tendency can be noticed that the number of sales is started to rise in the winter.
On the graph below it can be seen that in every city the total number of sales slowly droped at the beggining of autumn and then started to grow again at the beggining of winter. On this graph the difference between total number of sales in different cities is much clearer. This fact is not unusual, since the cities in Russia are quite disproportionate in population size and salary levels. Information about population size and salary levels was found here and here.

Lastly, there is a dependence between sale rates and weather conditions, although it is hard to interpret it properly, since some weather types are more frequent than others in different cities.
To build predictions of sales it is necessary to find data that correlates well with the number of sales. For that I converted the city name to numeric format (from 1 to 10), so that the higher number of sales can be seen in the city with the largest number of daily sales, Moscow, in my case. I added column with the number of the day of the week and another column indicating if it is the weekend or not. The weather description was converted to numeric format as well. Three columns were added with sales information from three previous weeks for each sale of the product from certain store. Another column was added where the mean of previous three weeks sales were calculated. Correlation matrix was obtained for the new dataframe. New columns show high correlation with the number of sales and should significantly improve predictions of future sales.
For further evaluation of ML models mean absolute error (MAE) is used as quality metric. MAE shows how many times the forecast is wrong on average. This metric is very easy to interpret. If we have a metric value equal to 5, this means that the model is on average wrong by 5. The metric cannot be negative, since Absolute values of errors are taken. For an ideal model this metric will be equal to 0. The metric is not sensitive to outliers.
$$MAE = \frac1N \sum ^{N}_{i=1} |y_i-\hat y_i|$$


Building ML models

Two dataframes were obtained (df_train, df_test), which contained all the modifications described previuosly. First three weeks in df_train were droped, since they have NAs in lag columns. First, several ML models were trained on df_train and the best model with minimal MAE score was chosen. For this, df_train was divided into 4 parts: y_train, x_train, y_test, x_test.

ML models:

  • k-nearest neighbors (KNN)
  • Linear Regression
  • Decision Tree Regression
  • Ensemble models
    • Random forest
    • Gradient Boosting
    • Average of all classifiers

Results

In this project, sales predictions were made for the stores across 10 russian cities for the 3rd weeek of February.
  • Exploratory analysis revealed patterns in data, which can be easily interpreted with russian population and salary distribution statistics.
  • Correlation between data and sales rates was improved by use one-hot encoding for object data, like city names.
  • Correlation was further improved by use of lag features of sales. The highest correlation was observed for mean number of sales for each day of the week for products in each store (day_product_mean) and mean of three lag days (lag_days_mean): 0.90 and 0.88 respectively.
  • 6 ML models were tested and evaluated using MAE score. The Decision Tree model had the lowest MAE score 3.07 and was chosen for predictions.
